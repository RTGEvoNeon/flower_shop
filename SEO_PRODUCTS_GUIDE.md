# Руководство по заполнению товаров для SEO-оптимизации

## Введение

Для корректной работы категорийных SEO-страниц необходимо правильно заполнять поля у товаров в базе данных. Эта документация объясняет, как использовать новые поля для фильтрации и SEO.

## Структура семантического ядра

У сайта есть 10 категорийных страниц с SEO-оптимизацией:

### Высокий приоритет (высокая частотность):
1. **Главная** (`/`) - "доставка цветов Брянск"
2. **Розы** (`/rozy`) - "купить розы Брянск"
3. **День рождения** (`/bukety-na-den-rozhdeniya`) - "букет на день рождения Брянск"
4. **8 марта** (`/bukety-na-8-marta`) - "букет на 8 марта Брянск"
5. **Недорогие букеты** (`/bukety-nedorogo`) - "букеты недорого Брянск"

### Средний приоритет (средняя частотность):
6. **Хризантемы** (`/hrizantemy`) - "купить хризантемы Брянск"
7. **Пионы** (`/piony`) - "купить пионы Брянск"
8. **В крафте** (`/bukety-v-krafte`) - "букет в крафте Брянск"
9. **Свадебные** (`/svadebnyy-buket`) - "свадебный букет Брянск"
10. **Композиции** (`/kompozicii-iz-cvetov`) - "композиции из цветов Брянск"

## Поля для заполнения в таблице products

### 1. flower_type (Тип цветов)

**Назначение:** Определяет основной тип цветов в букете.

**Допустимые значения:**
- `розы` - для букетов из роз
- `хризантемы` - для букетов из хризантем
- `пионы` - для букетов из пионов
- `тюльпаны` - для букетов из тюльпанов
- `лилии` - для букетов из лилий
- `смешанный` - для композиций из разных цветов
- `NULL` - если не применимо

**Примеры:**
```sql
-- Букет из 25 красных роз
UPDATE products SET flower_type = 'розы' WHERE id = 1;

-- Букет из кустовых хризантем
UPDATE products SET flower_type = 'хризантемы' WHERE id = 2;

-- Смешанный букет с розами и пионами
UPDATE products SET flower_type = 'смешанный' WHERE id = 3;
```

**Где отображается:**
- Страница `/rozy` покажет все товары с `flower_type = 'розы'`
- Страница `/hrizantemy` покажет все товары с `flower_type = 'хризантемы'`
- Страница `/piony` покажет все товары с `flower_type = 'пионы'`

---

### 2. occasion (Повод)

**Назначение:** Указывает, для какого события подходит букет.

**Допустимые значения:**
- `день рождения` - для букетов на день рождения
- `8 марта` - для весеннего праздника
- `свадьба` - для свадебных букетов
- `14 февраля` - для Дня святого Валентина
- `1 сентября` - для школьных букетов
- `юбилей` - для юбилеев
- `просто так` - для подарков без повода
- `NULL` - если не применимо

**Примеры:**
```sql
-- Яркий букет на день рождения
UPDATE products SET occasion = 'день рождения' WHERE id = 5;

-- Букет невесты
UPDATE products SET occasion = 'свадьба' WHERE id = 7;

-- Весенний букет тюльпанов
UPDATE products SET occasion = '8 марта' WHERE id = 10;
```

**Где отображается:**
- Страница `/bukety-na-den-rozhdeniya` покажет все товары с `occasion = 'день рождения'`
- Страница `/bukety-na-8-marta` покажет все товары с `occasion = '8 марта'`
- Страница `/svadebnyy-buket` покажет все товары с `occasion = 'свадьба'`

---

### 3. style (Стиль оформления)

**Назначение:** Определяет стиль упаковки или формат букета.

**Допустимые значения:**
- `в крафте` - для букетов в крафтовой бумаге
- `композиция` - для цветов в коробках, корзинах
- `классический` - для стандартных букетов
- `моно` - для монобукетов из одного вида цветов
- `каскад` - для свадебных каскадных букетов
- `NULL` - если не применимо

**Примеры:**
```sql
-- Розы в крафтовой упаковке
UPDATE products SET style = 'в крафте' WHERE id = 12;

-- Цветы в шляпной коробке
UPDATE products SET style = 'композиция' WHERE id = 15;

-- Монобукет из 101 розы
UPDATE products SET style = 'моно' WHERE id = 20;
```

**Где отображается:**
- Страница `/bukety-v-krafte` покажет все товары с `style = 'в крафте'`
- Страница `/kompozicii-iz-cvetov` покажет все товары с `style = 'композиция'`

---

### 4. price_range (Ценовой диапазон)

**Назначение:** Категоризация товаров по цене.

**Допустимые значения:**
- `недорого` - для букетов от 2000 до 3500 руб
- `средняя цена` - для букетов от 3500 до 6000 руб
- `премиум` - для букетов от 6000 руб
- `NULL` - если не применимо

**Примеры:**
```sql
-- Бюджетный букет за 2500 руб
UPDATE products SET price_range = 'недорого' WHERE id = 25;

-- Роскошный букет за 8000 руб
UPDATE products SET price_range = 'премиум' WHERE id = 30;
```

**Где отображается:**
- Страница `/bukety-nedorogo` покажет все товары с `price_range = 'недорого'`

---

### 5. tags (Теги, JSON)

**Назначение:** Массив дополнительных тегов для фильтрации и поиска.

**Формат:** JSON массив строк

**Примеры тегов:**
- Цвета: `красный`, `белый`, `розовый`, `желтый`, `микс`
- Размеры: `маленький`, `средний`, `большой`, `гигантский`
- Особенности: `ароматный`, `гипоаллергенный`, `сухоцветы`
- События: `для мамы`, `для коллеги`, `для любимой`

**Примеры:**
```sql
-- Красные розы для романтического свидания
UPDATE products SET tags = '["красный", "большой", "ароматный", "для любимой"]' WHERE id = 35;

-- Нежный букет для мамы
UPDATE products SET tags = '["розовый", "средний", "для мамы"]' WHERE id = 40;
```

---

### 6. SEO-поля: meta_title, meta_description, meta_keywords

**Назначение:** Оптимизация для поисковых систем. Эти поля заполняются автоматически, но можно переопределить.

**meta_title** - заголовок страницы товара (до 60 символов)
**meta_description** - описание страницы товара (до 160 символов)
**meta_keywords** - ключевые слова (до 250 символов)

**Примеры:**
```sql
UPDATE products SET
    meta_title = 'Букет из 25 красных роз в Брянске | Доставка — Mindale',
    meta_description = 'Классический букет из 25 роз с доставкой по Брянску. Свежие эквадорские розы в красивой упаковке. Цена 4500₽',
    meta_keywords = 'букет роз, красные розы, 25 роз, доставка роз Брянск'
WHERE id = 50;
```

## Массовое обновление через SQL

Если у вас уже есть товары в базе, можно обновить их массово:

```sql
-- Все розы отметить как "розы"
UPDATE products
SET flower_type = 'розы'
WHERE name LIKE '%роз%' OR description LIKE '%роз%';

-- Все товары дешевле 3500 руб отметить как недорогие
UPDATE products
SET price_range = 'недорого'
WHERE price <= 3500;

-- Все товары дороже 6000 руб отметить как премиум
UPDATE products
SET price_range = 'премиум'
WHERE price >= 6000;

-- Все товары в среднем диапазоне
UPDATE products
SET price_range = 'средняя цена'
WHERE price > 3500 AND price < 6000;
```

## Проверка заполнения

Проверьте, сколько товаров у вас в каждой категории:

```sql
-- Товары по типам цветов
SELECT flower_type, COUNT(*) as count
FROM products
WHERE is_available = 1
GROUP BY flower_type;

-- Товары по поводам
SELECT occasion, COUNT(*) as count
FROM products
WHERE is_available = 1
GROUP BY occasion;

-- Товары по стилю
SELECT style, COUNT(*) as count
FROM products
WHERE is_available = 1
GROUP BY style;

-- Товары по ценовому диапазону
SELECT price_range, COUNT(*) as count
FROM products
WHERE is_available = 1
GROUP BY price_range;
```

## Рекомендации

1. **Один товар может отображаться на нескольких страницах** - это нормально и даже желательно для SEO
2. **Заполняйте все релевантные поля** - чем больше информации, тем лучше для поиска
3. **Используйте точные значения** - пишите "розы", а не "Розы" или "роза"
4. **Заполняйте tags** - это поможет в будущем добавить фильтры на сайте
5. **Проверяйте цены** - `price_range` должен соответствовать реальной цене товара
